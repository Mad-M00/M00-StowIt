[b][size=4]v1.3.34 (July 2026)[/size][/b]

[b]Bug fixes[/b]
[list]
[*]Sorting now works alongside automation mods like Overengineered. Those mods hook the game's "put item into container" routines for their pipe networks, which could leave a sort finding your chests but moving nothing; StowIt now places items into the crate directly, the same reliable way restock already reads from it.
[*]Sorting reports its result in the console ("moved X item(s) into Y of Z crate(s)"), and a container lock that is refused says so - so if anything ever blocks a sort again, the log will show exactly where it stopped.
[/list]

[b]Documentation[/b]
[list]
[*]New FAQ and install-page answer: yes, you can add StowIt to an existing save. Containers keep their names, unrecognised sign text is simply ignored, and nothing needs renaming or emptying first.
[/list]

[b][size=4]v1.1.33 (July 2026)[/size][/b]

[b]Bug fixes[/b]
[list]
[*]Sorting now plays nicely with paged backpack mods like Adaptive Backpack. Previously their "stash all" takeover swept your whole backpack into the first crate; StowIt now moves every item itself, so sorting routes correctly with or without those mods installed. With a paged backpack, sorting works on the page you are looking at.
[/list]

[b][size=4]v1.1.30 (July 2026)[/size][/b]

[b]Bug fixes[/b]
[list]
[*]The blue box some players saw when restocking is gone. It was the game's own editor selection tool reacting to the Z in LeftAlt + Z (it only shows up when the debug menu is on). StowIt's hotkeys no longer trigger the game's editor keys - and if you ever see the box from pressing plain Z yourself, Backspace clears it.
[*]Restocking no longer plays the container-open sound for every crate it pulls from.
[*]Slots you lock inside a crate now stay put when you restock, and locked slots from the last container you looted no longer quietly block restocking.
[/list]

[b]Documentation[/b]
[list]
[*]New FAQ answer: yes, you can change the key bindings - any keys you like, via StowItConfig.xml. The FAQ points you to the step-by-step guide.
[/list]

[b][size=4]v1.0 - Initial release (July 2026)[/size][/b]

[b]Sorting[/b]
[list]
[*]One keypress ([b]LeftAlt + X[/b]) sorts your whole backpack into nearby Writable Storage Crates by the word on their sign.
[*]26 category names work out of the box: Ammo, Food, Medical, Tools, Books, Resources, Farming, Building, Weapons and more.
[*]Forgiving sign matching: case, punctuation and plurals never matter. "Mod Tools", "Mods \ Tools" and "MOD-TOOLS" are the same.
[*]Specific crates beat general ones: an "Ammo 9mm" crate wins over "Ammo" automatically.
[*]Overflow chaining: same sign on two crates? Nearest fills first, the rest walks outward crate by crate.
[*]A crate signed "Misc" catches everything unmatched - nothing is ever lost or destroyed.
[*]Containers without a category sign still receive top-ups of items they already hold.
[/list]

[b]Restock[/b]
[list]
[*][b]LeftAlt + Z[/b] refills your carried stacks (ammo, food, meds...) from nearby crates before you head out.
[/list]

[b]What it never touches[/b]
[list]
[*]Your toolbelt, worn armor and equipped items.
[*]Backpack slots locked with the game's own lock button.
[/list]

[b]Customising[/b]
[list]
[*]All rules live in one plain text file (CrateLabels.txt): groups, item names, wildcard patterns like foodCan*, and exclusions with a minus sign.
[*]Edit rules from inside the game with the stow console command: stow alias, stow what, stow search, stow groups, stow reload.
[*]Every in-game edit keeps an automatic backup (CrateLabels.txt.bak).
[*]Hotkeys are configurable in StowItConfig.xml.
[/list]

[b]Languages[/b]
[list]
[*]Sign labels ship for all official game languages: de, es, fr, it, ja, ko, pl, pt-BR, ru, tr, zh-CN, zh-TW.
[/list]

[b]Compatibility[/b]
[list]
[*]Items from other mods sort like vanilla ones - the item list is read from the running game.
[*]Multiplayer and dedicated server safe: uses the game's own container locking, so no lost or duplicated items.
[*]EAC must be off (like any DLL mod); with EAC on the mod safely skips itself.
[*]Uninstall any time by deleting the mod folder - saves are never touched.
[/list]
